### 阶段3：代码讲解（3-4分钟）

```
切换窗口：Alt+Tab 切回 Visual Studio
确保解决方案资源管理器在左侧显示所有文件

═══════════════════════════════════════
代码讲解1：继承与虚函数（约1.5分钟）
═══════════════════════════════════════

1. 在解决方案资源管理器双击 User.h 打开
   鼠标指向第 15-18 行：
   "这四个纯虚函数 = 0 强制所有派生类必须实现
    ——display、getMaxBorrow、getBorrowDays、getRole"

2. 双击 Student.h，指向第 20 行：
   "override 关键字——编译器会检查是否正确重写了基类虚函数"

3. 双击 Student.cpp，指向第 22-26 行：
   "getMaxBorrow 返回 5，getBorrowDays 返回 30"

4. 双击 Teacher.cpp，指向第 22-26 行：
   "同样的函数，Teacher 返回 10 和 60
    ——不同类型的读者自带不同的规则"

═══════════════════════════════════════
代码讲解2：多态简化逻辑（约1.5分钟）
═══════════════════════════════════════

5. 双击 Library.cpp
   用 Ctrl+G 跳到第 331 行（borrowItem 函数）

6. 指向第 345 行 findUser：
   "findUser 返回 User* 类型——它不知道返回的
    是 Student 还是 Teacher"

7. 指向第 353 行 u->getMaxBorrow()：
   "调用 getMaxBorrow()——Student 自动返回 5，
    Teacher 自动返回 10。Library 不需要判断类型
    ——这就是多态最核心的价值"

8. 指向第 373 行 u->getBorrowDays()：
   "同样，借阅天数也是多态——Student 30天，
    Teacher 60天"

═══════════════════════════════════════
踩坑经历（约1分钟）
═══════════════════════════════════════

9. 跳到 deleteUser 函数（第273行附近）：
   "最初 deleteUser 只检查了未归还的记录就放行删除。
    但已归还的历史记录里还持有指向被删 User 的指针
    ——悬空指针，之后查看记录就崩溃"

   "怎么发现的？析构函数里加日志打印每个地址，
    发现崩溃地址和已 delete 的地址匹配——二次释放"

   "修复方案：第283行——删除前检查所有借阅记录，
    只要有就拒绝删除。虽然不能删有关联记录的实体，
    但总比崩溃好"
```
