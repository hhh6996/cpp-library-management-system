### 阶段 3：代码讲解（3-4 分钟）

```
Alt+Tab 切回 VS2022

══════ 继承与虚函数 ══════

1. 打开 User.h → 指向第 15-18 行
   "四个纯虚函数 = 0——所有派生类必须实现"

2. 打开 Student.cpp → 指向第 22-26 行
   "getMaxBorrow 返回 5，getBorrowDays 返回 30"

3. 打开 Teacher.cpp → 指向第 22-26 行
   "同样的函数，Teacher 返回 10 和 60"

══════ 多态 ══════

4. 打开 Library.cpp → Ctrl+G 跳到第 331 行
   "borrowItem 函数——理解多态的最佳入口"

5. 指向第 345 行 findUser：
   "findUser 返回 User*——不知道是 Student 还是 Teacher"

6. 指向第 353 行 u->getMaxBorrow()：
   "调用 getMaxBorrow——Student 自动返回 5，Teacher 返回 10。
    Library 一行 if-else 都没写——这就是多态"

7. 指向第 373 行 u->getBorrowDays()：
   "借阅天数同理——Student 30天，Teacher 60天"

══════ 踩坑 ══════

8. 跳到 deleteUser（约第 273 行）：
   "最初这里只检查未归还记录就放行删除——但已归还的历史记录
    还指着被删的 User，之后再查看记录就崩溃——悬空指针"

   "怎么发现的？析构函数加日志打印地址，发现崩溃地址
    和已删除地址匹配——二次释放"

   "修复：第 283 行——检查所有记录，有就拒绝删除"
```
